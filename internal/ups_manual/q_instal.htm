1.インストール


OSの種類、バージョンまたはコンピュータの機種によって、本マニュアルが実際と食い違う場合がありますのでご注意ください。

ワークステーションとUPSの接続

  1. ワークステーションおよびUPSの電源を切ります。
  2. ワークステーションの電源コードをUPS背面の出力コンセントに接続します。
  3. 付属のインタフェースケーブルを使って、ワークステーションのシリアルポート （RS-232C）とUPSのコミュニケーションコネクタを接続します。
Sunワークステーションでttybを使用する場合、一部の機種では オプションケーブルが必要です。
詳しくはメーカにお問い合わせください。
HP9000のEthernetインタフェースは終端抵抗を内蔵していません。
HPワークステーションを単体で使用する場合は、必ず外部終端抵抗を付けてください。

インストールの前に

正しくUPSとワークステーションを接続してください。 また、UPSを接続するシリアルデバイス（tty）を確認してください。シリアルデバイスがない場合は作成する必要があります。作成方法は、OSのドキュメントを参照してください。
OSの種類によってインストール方法が異なります。ご使用のOSとそのバージョンを確認の上、以下の説明に従って、OSごとに必要な設定を行ってください。

  1. HPの場合
    HP9000シリーズではUPS制御ポートをUPSとの接続に利用できます。その場合には、 /etc/inittabファイルの次の行をコメントアウトしてください。行がない場合は、 変更の必要がありません。
    # ups::respawn:rtprio 0 /etc/ups_mond -f /etc/ups_conf

  2. SiliconGraphicsの場合
    IRIX6.3以降のOSでは、使用するシリアルポートの設定が「available」になって いることを確認してください。

  3. BSDIの場合
    Version3以降ではイベントログデータが保存されません。
    これは、syslogdが通信で使用するUNIXドメインのソケット名が/dev/logから /var/run/logに変更になったためです。/dev/logの名前でシンボリックリンク を作成してください。
    # ln -s /var/run/log /dev/log <Return>

インストールプログラムのコピー

インストール用ファイルとして、TAR形式ファイルとRPM形式ファイルが用意されています。

１．TAR形式ファイルを使用する場合

CD-ROMからプログラムを、以下の手順でワークステーションの"/usr"ディレクトリにコピーします。

インストールは、スーパユーザ（ルート権限のある人）が行う必要があります。
  1. ルートでログインします。
    root<Return>
  2. カレントディレクトリを"/usr"に変更します。 "/usr"のパーティションに空きがなければ任意のディレクトリにインストールしてもかまいません。
    # cd /usr<Return>
  3. CD-ROMをドライブに装着して、ファイルシステムとしてマウントします。
    マウント方法は、OSやバージョンによって変わります。 詳しくはOSのマニュアルを参照してください。
  4. 必要なプログラムをカレントディレクトリにコピーします。
    カレントディレクトリの下にupsというディレクトリが作られて、その下にプログラムがコピーされます。
    # tar xvf filename <Return>

２．RPM形式ファイルを使用する場合

CD-ROMからプログラムを、以下の手順でワークステーションの"/usr"ディレクトリにコピーします。

コピーは、/usrディレクトリの下にupsというディレクトリを作成します。
RPM形式でのインストールはインストール先の変更には対応しておりません。
インストール先を任意の場所に変更してインストールする場合には、ＴＡＲ形式でのインストールを実行してください。
インストールは、スーパユーザ（ルート権限のある人）が行う必要があります。
  1. ルートでログインします。
    root<Return>
  2. カレントディレクトリを"/tmp"に変更します。 "/tmp"のパーティションに空きがなければ任意のディレクトリにコピーしてもかまいません。
    # cd /tmp<Return>
  3. CD-ROMをドライブに装着して、ファイルシステムとしてマウントします。
    マウント方法は、OSやバージョンによって変わります。 詳しくはOSのマニュアルを参照してください。
  4. 必要なプログラム(RPM)をカレントディレクトリにコピーします。
    カレントディレクトリの下にRPMファイルがコピーされます。
    # tar xvfz filename <Return>

    ※ filenameは使用環境とインストールOSによってかわります。下表を参考にしてください。

    Linux(Kondara MNU/Linux) Intel版 v3/free/i386/rpm/pv_kond.taz
    Linux(OpenLinux, eServer) Intel版 v3/free/i386/rpm/pv_cald.taz

  1. 5.のコマンドを実行すると、/usrディレクトリの下にups以下のディレクトリが作成されます。
  1. RPMのインストールが終了したら、「ソフトウエアのインストール」に進んでください。
